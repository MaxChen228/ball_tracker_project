# 00 — Research Synthesis (2026-05-01)

階段性整合。基於 1073 SAM2-labeled GT frames / 9 sessions。

## TL;DR

| 路線 | R | 成本 | 狀態 |
|---|---|---|---|
| V11 (HSV+CC) | 0.905 | ~1.0 ms iPhone C++ | shipped |
| V11 + Y-diff (thr=15) | **0.970** | ~1.5 ms | **新 best clean baseline** |
| V11 + Y-diff + ROI-FRST | 0.9739 | ~3 ms | +0.37pp，邊際 |
| V11 + full-FRST (no ROI) | 0.996 | 不可用 | FP 9789/frame |
| Tiny FCN distilled (LOSO) | 0.105 | n/a | 1073 frames 不夠訓 |

**從研究開始的 0.905「物理 ceiling」被打破到 0.970**，主要靠 Y-plane 時間差分（cue independence MI=0.009，與 V11 幾乎正交）。

## 已 lock-in 的研究結論

### 物理機制

1. **V11 失敗模式分為三類**：M1 specular (n=68, 球反白光顏色喪失) / M2 fragmentation (n=24) / M3 hue shift (n=9)
2. **Mode α 的物理本質是 ISP-clipped pure white**：68 個 M1 frame 的 GT region BGR 三通道差 <5/255，已是純灰白。dichromatic 分離不適用
3. **V11 用 chrominance 當 spatial gate；Y-diff 用 motion 當 spatial gate** — 兩個 cue 在資訊論上 mutual information 0.009 bits（NMI 0.019），near-maximally independent
4. **240 fps 下球位移 50-300 px、bg 靜止** — Y-channel |Δ| 是高 SNR motion blob，與顏色完全脫鉤，TrackNetV4 (arXiv:2409.14543) + ACM MMSports 2023 已用同概念

### 演算法極限

5. **多尺度 Y-diff (D1+D2+D3) union ceiling = 0.9842**（Track S）— mode-routed oracle 反而 0.9720（routing decision 引入錯誤）。對 binary detectors 不需要 failure-mode classifier。**Apex 假設 falsified**：D2/D3 不在慢球/apex 幀補強，反而在 fast 幀（D3 vs D1 +16.6pp）— 大 baseline 需要大位移才形成 clean blob，慢球幀 t 與 t-3 位置重疊自我抵銷
6. **21 consensus residual frames = single-frame stateless death zone**（Track V 已完成）— 分 4 群：G1 edge-clipped (n=5, FoV flag) / G2 specular setup run (n=6, sensor clipped, capture-side fix) / G3 low-contrast mid-flight (n=5, gap-fill) / G4 mixed late-flight (n=5, gap-fill)。20/21 isolated miss → trajectory gap-fill 救 G3+G4 = **+0.93pp，零新 model**

### 平台

7. **iOS 現況偵測 ~15 ms**（不是 1 ms）— Phase 1 必須先 parallelize 現有 detector
8. **A15 ANE 上 depthwise conv 比 Metal GPU 慢 5×** — MobileNet/EfficientNet 系列直接放 ANE 是 anti-pattern
9. **tiny standard-conv FCN @ 192-256 input <500K params 在 A15 ANE 5-8 ms 可行**（YOLOv5s @ 192×320 on A14 = 10 ms 為 anchor）
10. **`VNDetectCirclesRequest` 不存在** — Apple Vision 無原生 circle detection
11. **`VNDetectTrajectoriesRequest`（iOS 14+）真實存在** — 偵測拋物線軌跡，可作 post 路徑 free verifier
12. **`YOLOv10/v11 1-5ms FP16 ANE` 是錯標**：原數字是 T4 GPU TensorRT。真機 ~12-20 ms FP16 / ~3-6 ms INT8 only on A17 Pro+

## 已 falsify 的方向（負結果有研究價值）

| 方向 | 失敗原因 | 收窄了什麼 |
|---|---|---|
| Dichromatic separation (Yang 2010 / Tan 2005) | ISP 已 clip 訊號到純灰白，模型前提不成立 | 整個 dichromatic 家族退役 |
| Tiny FCN from-scratch on 1073 frames | dense regression imbalance + 資料量太少 → loss vanish | 1073 frames 不足以從 scratch 訓 DL |
| Cue-consistency aux loss for distillation | FRST teacher recall 4% 為 FP 主導，aux loss 學不到信號 | distillation 需更乾淨 teacher |
| CLAHE / S-stretch global pre-process | 22d1835e_b -72.73pp、22d1835e_a -17.54pp 災難 | 全域 contrast enhance 破壞 spatial isolation |
| Top-K candidate cap | K=1,2,3,5 都只救 3 frame | bg distractors 主導 area top-K |
| Stateful ROI-FRST | ROI gating 依賴 recent detection；最難的 21 frames 沒 detector fire = ROI 空 | ROI gate 解不了 consensus residuals |
| Single-frame Mode α algorithmic rescue | M1 frame 球已是 pure-white，無 chromatic info 可挖 | Mode α 真正解法在 capture stage |

## 研究上的下一個真問題

**真實 ceiling 在哪？**

- **Single-frame ceiling**：簡單 OR 三 cue = 0.9804。要破得進入 multi-frame
- **Multi-frame stateless ceiling**：未測。Multi-scale Y-diff (Track S) + VNDetectTrajectoriesRequest 是兩個 candidate
- **Trajectory gap-fill ceiling (新)**：Track V 證明 G3+G4 共 10 frame 用 ballistic spline + Kalman 即救 → R=0.970→0.979。G2 排除後 R=0.985。這是 viewer-side post-processing 即可達成，不需新 detector
- **Multi-frame stateful ceiling**：track + Kalman + ballistic prior，預計 0.99+ 但需要 stateful 設計
- **Capture-improved ceiling**：CPL / ND filter / 曝光鎖定 後 Mode α 應該物理消失
- **DL ceiling on this dataset size**：證實 1073 不夠。10K+ 後重啟

## 研究價值提取

這輪研究產出的不只是「R 從 0.905 推到 0.970」這個數字，更重要的是：

1. **以資訊論證明兩個 cue 物理上正交**（MI=0.009）— 這是 V11+Y-diff fusion work 的根本理由，不是 ad-hoc
2. **以 1073 frame 量化 dichromatic 模型在 ISP-pipeline 後的失效**— 不只在我們資料失敗，告訴所有用 iPhone capture 的研究這條路都不通
3. **以 LOSO 失敗證明 1073 frames 是 small-data DL boundary**— 給未來資料規劃明確 ROI 估算
4. **修正多個常見 LLM-generated 誤解**（VNDetectCirclesRequest, YOLOv10 5ms）— 給未來 agent / 顧問 fact-check checklist

研究的「方法」比「數字」更值得 preserve — 派 agents 深查 + 物理量化 + cue 獨立性分析這套 protocol 可以重複用在下一個問題。
